Welcome to Ether Fund Raiser.

This DApp allows you to raise cryptocurrency (Ether) funds using Smart Contracts deployed on the Ethereum Blockchain. Once the goal of the contract has been reached, the contract owner can then issue a Spending Request to release the funds which then requires over 50% of the contributors to approve before the Ether is released. Any contracts failing to reach their goal by the deadline can be refunded. Also, any contracts that reached their goal but have no approved and paid spending requests within an initial payment period after the deadline can also be refunded.

To use this DApp you will need an Ethereum Wallet installed in your browser, such as MetaMask, and give this DApp permission to connect to your wallet. Any transactions you initiate that require processing will display the MetaMask approval window prior to forwarding to the blockchain. You can use MetaMask to connect to the various Test Networks in order to test this DApp with test Ether before switching to the Main Ethereum Network (see examples below). An [Events] section is included at the bottom of the screen to communicate all processing between this DApp and the blockchain. By clicking on the transaction or contract address hyperlinks you can use Etherscan to see the actual status and transaction history as recorded on the blockchain.

How it works - For Fund-Raisers.

Start by choosing the [New Contract] menu option to deploy a new fund-raising smart contract, selecting the duration of the contract (in blocks of roughly 15 seconds), the initial payment period (also in blocks) after the deadline to allow payments to be approved and released, the financial goal and the minimum contribution level (both in Wei - the smallest denomination of Ether). Once deployed, you can email a link of the deployed contract to your contributors. There is a [Wei Converter] to assist in converting values between Wei, Ether and USD, and a [Block Converter] to assist with calculating (and resetting) block times, which defaults to the current network average block time.

From the [Existing Contract] menu option you and your contributors can see the status of the deployed fund-raising smart contract and can make contributions. As the owner you can check the contribution made from a contributor's account, and also change the ownership of the contract to a different account.

Once the smart contract has reached its goal, you can use the [Spending Requests] menu option to create one or more Spending Requests (up to a maximum of ) outlining how the funds will be spent, and then email a link of the spending request to your contributors. The contributors can then vote on the request and you can check if a specific contributor's account has voted. Only when over 50% of contributors have voted can the amount requested be released via the [Pay Request] button.

How it works - For Contributors.

You will have been sent the Ethereum address of the relevant fund-raising Smart Contract you wish to support. From the [Existing Contract] menu option, enter this address and click on [Get Details] to see the status of the deployed fund-raising smart contract. You can then use the [Contribute] button to make contributions at or above the minimum contribution amount. If the smart contract fails to raise the Goal amount before the end of the deadline then you can use the [Refund] button to receive a refund of any funds you have contributed. You can also get a refund if the contract reached its goal but has not had any spending requests approved and paid within the initial payment period after the deadline.

Once the smart contract has reached its goal, the owner will create one or more Spending Requests. These can be viewed using the [Spending Requests] option where you can enter the request ID and click on [Get Details] to display the request detailing how the money will be spent. As a contributor you can then vote on each request and only when over 50% of contributors have voted for a request can the relevant payment be released.

Important Notes.

The blockchain is constantly updating (for example there is a new block mined roughly every 15 seconds) and this DApp only provides a snapshot of data that is updated with each interaction. If left idle for too long, the data shown will become out-of-date. To get the latest information click again on [Get Details].

Each fund-raising Smart Contract is "owned" by the account used during the deployment. Certain owner-only actions (such as creating and paying Spending Requests) can ONLY be actioned by the "owner". Therefore, if the wallet or keys of the owner account are lost there is no way to recover these funds. There are no secret backdoors to these contracts!

All transactions connected with this site are performed directly on the selected blockchain. The miners will charge minimal standard fees for each transaction processed, including deploying the contract, making a contribution, voting for a spending request, releasing payment of a request, etc. NO additional fees are charged, and NO fees are deducted from the amounts raised. However, any transaction fees paid to the miners are non-refundable. To make a contribution towards the running of this site please send any donations to: .
